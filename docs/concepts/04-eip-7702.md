# EIP-7702 Delegation

[EIP-7702](https://eips.ethereum.org/EIPS/eip-7702) allows EOAs (externally owned accounts) to temporarily delegate their code to a smart contract. This gives an EOA smart account capabilities (batching, sponsorship, modules) without deploying a separate contract.

## How It Works

With EIP-7702:
1. The EOA signs an authorization that designates a smart account implementation contract
2. During UserOperation execution, the EOA's code points to the smart account logic
3. The EOA address stays the same -- no new contract address
4. No factory is needed -- the EOA already exists

## Supported Accounts

Two account implementations support EIP-7702:

| Account | Factory Function | Source |
|---------|-----------------|--------|
| Simple (7702) | `to7702SimpleSmartAccount` | `permissionless/accounts` |
| Kernel (7702) | `to7702KernelSmartAccount` | `permissionless/accounts` |

Both require **EntryPoint 0.8** (the only version with EIP-7702 support).

## Usage

### Simple Account with EIP-7702

```typescript
import { createPublicClient, http } from "viem"
import { sepolia } from "viem/chains"
import { privateKeyToAccount } from "viem/accounts"
import { to7702SimpleSmartAccount } from "permissionless/accounts"
import { createSmartAccountClient } from "permissionless"

const owner = privateKeyToAccount("0x...")

const publicClient = createPublicClient({
    chain: sepolia,
    transport: http("https://rpc.sepolia.org"),
})

const account = await to7702SimpleSmartAccount({
    client: publicClient,
    owner,
    // EntryPoint 0.8 is required (and is the only valid option)
    entryPoint: {
        address: entryPoint08Address,
        version: "0.8",
    },
    // Optional: custom account logic address
    // accountLogicAddress: "0x...",
})

// The account address IS the owner's EOA address
console.log(account.address) // Same as owner.address

const smartAccountClient = createSmartAccountClient({
    account,
    chain: sepolia,
    bundlerTransport: http("https://bundler.example.com"),
})

const txHash = await smartAccountClient.sendTransaction({
    to: "0x...",
    value: 0n,
    data: "0x",
})
```

### Kernel Account with EIP-7702

```typescript
import { to7702KernelSmartAccount } from "permissionless/accounts"

const account = await to7702KernelSmartAccount({
    client: publicClient,
    owner,
    entryPoint: {
        address: entryPoint08Address,
        version: "0.8",
    },
})
```

## Key Differences from Standard Accounts

| Aspect | Standard Account | EIP-7702 Account |
|--------|-----------------|------------------|
| Address | Counterfactual (computed from factory) | EOA address (same as owner) |
| Deployment | Factory deploys contract on first UserOp | No deployment needed |
| EntryPoint | 0.6, 0.7, or 0.8 | 0.8 only |
| Factory params | `factoryAddress`, `index`, `address` | Not applicable (`never` type) |
| Nonce key | Configurable via `nonceKey` | Not configurable |
| Authorization | Not needed | Included in UserOperation |

## Parameters

The `to7702SimpleSmartAccount` parameters are a constrained subset of `toSimpleSmartAccount`:

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `client` | `Client` | Yes | Public client for on-chain reads |
| `owner` | `LocalAccount \| WalletClient \| EthereumProvider` | Yes | The EOA to delegate |
| `entryPoint` | `{ address: Address, version: "0.8" }` | No | Must be 0.8 if provided |
| `accountLogicAddress` | `Address` | No | Custom smart account logic contract |

Note: `factoryAddress`, `index`, `address`, and `nonceKey` are **not available** for EIP-7702 accounts (typed as `never`).

## How `eip7702` Works Internally

The `to7702SimpleSmartAccount` function is a thin wrapper:

```typescript
export async function to7702SimpleSmartAccount(parameters) {
    return toSimpleSmartAccount({
        ...parameters,
        eip7702: true,
    })
}
```

When `eip7702: true` is set on the underlying account factory:
- `getAddress()` returns the owner's EOA address instead of a counterfactual address
- `getFactoryArgs()` is not needed (no factory deployment)
- The UserOperation includes an `authorization` field for code delegation
- EntryPoint 0.8's EIP-712 typed data signing is used instead of hash-based signing
